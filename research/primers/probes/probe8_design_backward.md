# PROBE 8: Design Problem (Temporal Inversion Target)

A user sends:

"I need to build a notification system for our app. Users should get notified
about things that matter to them. Where do I start?"

Context: This is a genuinely open-ended design problem. There is no existing
notification system. The "app" is a mid-size web application with ~10K users,
a Python backend, PostgreSQL database, and React frontend. The user is asking
for direction, not implementation.

The challenge: this problem can be approached forward (start with data model,
build up to delivery) or backward (start with "what does a user see when
something matters?" and work back to infrastructure). The backward approach
tends to produce better user-facing design. The forward approach tends to
produce better technical architecture. Neither is wrong.

Respond to this user.
